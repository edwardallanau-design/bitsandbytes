# Payload CMS Schema for Bits & Bytes

## Collections

### `projects` — Work showcase items
```
- title (text, required)
- slug (text, unique, auto-populate from title)
- year (number)
- client (text)
- kind (select: "Web app", "iOS app", "Website", "Design system")
- category (text: e.g., "Engineering", "Design", "Product")
- summary (textarea)
- description (richText)
- tags (array of strings)
- featured (checkbox) — show on homepage
- order (number) — sort by this
- imageUrl (upload or text URL) — hero image for work card
```

### `testimonials` — Social proof
```
- quote (textarea, required)
- author (text, required)
- role (text) — "A&R, Universal Music Group"
- avatarInitials (text, max 2)
- featured (checkbox)
```

### `pricing-tiers` — Pricing section
```
- name (text: "Starter", "Professional", "Enterprise")
- description (text)
- price (text: "$999", "Custom", etc.)
- cadence (text: "one-time", "per month", empty for Enterprise)
- featured (checkbox)
- features (array of text items)
- order (number)
```

### `team-members` — About section
```
- name (text, required)
- role (text: "Lead Developer", "Designer", etc.)
- initials (text, max 2)
- bio (textarea, optional)
- avatar (upload, optional)
- order (number)
```

### `blog-entries` (optional) — Future journal / news
```
- title (text, required)
- slug (text, unique)
- date (date)
- body (richText)
- excerpt (textarea)
- tags (array of strings)
- published (checkbox)
```

### `site-settings` (global config)
```
- availabilityStatus (text: "Currently booking · Q1 2026")
- heroTitle (textarea: "Software solutions, shipped fast.")
- heroSubtitle (textarea)
- aboutPillText (text: "Accepting new projects")
- contactEmail (email)
- contactPhone (text)
```

## API Endpoints (auto-generated by Payload)

- `GET /api/projects` — list all (paginated)
- `GET /api/projects?where[featured][equals]=true` — featured only
- `GET /api/projects/:id` — single project detail
- `GET /api/testimonials?limit=100` — all testimonials
- `GET /api/pricing-tiers?sort=order` — sorted pricing
- `GET /api/team-members?sort=order` — sorted team
- `GET /api/site-settings` — global config
- `POST /api/projects`, `PATCH /api/projects/:id`, `DELETE /api/projects/:id` — CMS team use

## Notes

- All collections have `createdAt`, `updatedAt` auto-fields.
- Use **Access Control** to restrict editing to authenticated team members.
- **Hooks** can auto-generate slugs, image alt-text, etc.
- **Vercel/Netlify** can trigger rebuilds on `POST` hooks if using static generation.
