# Bits & Bytes + Payload CMS Integration

## Quick Start

### 1. Install Payload
```bash
npm install payload express dotenv
```

### 2. Create `.env`
```
MONGODB_URI=mongodb://localhost:27017/bits-and-bytes
PAYLOAD_SECRET=your-secret-key-here
```

### 3. Set up MongoDB locally
```bash
# macOS with Homebrew
brew services start mongodb-community

# Or use Docker
docker run -d -p 27017:27017 --name mongo mongo:latest
```

### 4. Copy the collections
```
payload/
  ├─ collections/
  │  ├─ Users.ts          (auto-generated by Payload)
  │  ├─ Projects.ts       (copy from this repo)
  │  ├─ Testimonials.ts   (copy from this repo)
  │  ├─ PricingTiers.ts
  │  └─ TeamMembers.ts
  ├─ globals/
  │  └─ SiteSettings.ts
  └─ payload.config.ts
```

### 5. Run Payload
```bash
npm run payload dev
# Admin panel at http://localhost:3000/admin
# API at http://localhost:3000/api
```

### 6. Update the website to fetch from Payload

In `ui_kits/website/`:
- Import the `useFetchPayload` hook from `hooks.ts`
- Replace hardcoded data with API calls
- Example: `WorkGrid.example.tsx` shows the pattern

## API Usage in the marketing site

**Fetch all projects:**
```javascript
const res = await fetch('/api/projects?sort=-order');
const { docs } = await res.json();
```

**Fetch featured projects:**
```javascript
const res = await fetch('/api/projects?where[featured][equals]=true');
const { docs } = await res.json();
```

**Fetch pricing tiers (sorted):**
```javascript
const res = await fetch('/api/pricing-tiers?sort=order');
const { docs } = await res.json();
```

**Fetch site-wide settings:**
```javascript
const res = await fetch('/api/globals/site-settings');
const settings = await res.json();
```

## Access Control

By default, `GET` requests are public (anyone can read). Authenticated team members can edit via `/admin`.

To restrict reads to authenticated users, add to each collection:
```typescript
access: {
  read: ({ req: { user } }) => !!user,
}
```

## Deployment

- **Vercel:** Deploy the Next.js app normally. Set `MONGODB_URI` and `PAYLOAD_SECRET` in env vars.
- **Netlify:** Use serverless functions for API calls, or deploy Payload separately (Railway, Render, etc.).
- **Self-hosted:** Run both on the same VPS (Payload on port 3000, Next.js on 3001).

## Rebuilding the site on content changes

Add a **post-change hook** in Payload to trigger a rebuild:
```typescript
hooks: {
  afterChange: [
    async () => {
      // Trigger Vercel rebuild
      await fetch('https://api.vercel.com/v1/deployments', {
        method: 'POST',
        headers: { Authorization: `Bearer ${process.env.VERCEL_TOKEN}` },
        body: JSON.stringify({ projectId: '...' }),
      });
    },
  ],
},
```
